# 📄 Brief projet — SportBook

## 🏀 1. Présentation du projet

SportBook est une application web de réservation de séances sportives en ligne.  
Elle permet aux utilisateurs de consulter des séances de sport disponibles et de réserver une place en quelques clics.

L’application inclut également une interface d’administration permettant de gérer les séances et les réservations.

---

## 🎯 2. Objectifs du projet

- Permettre la réservation de séances sportives en ligne
- Faciliter la gestion des utilisateurs et des séances
- Offrir une interface simple, responsive et accessible
- Appliquer les bonnes pratiques de développement web
- Mettre en place une base de données relationnelle sécurisée

---

## 👥 3. Public cible

- Utilisateurs souhaitant réserver des activités sportives
- Administrateurs de salle de sport ou coachs sportifs

---

## ⚙️ 4. Fonctionnalités principales

### 👤 Utilisateur
- Inscription / connexion
- Consultation des séances disponibles
- Réservation d’une séance
- Annulation d’une réservation
- Consultation de son profil et de ses réservations

### 🛠️ Administrateur
- Création de séances sportives
- Modification et suppression de séances
- Gestion des utilisateurs et réservations

---

## 🗄️ 5. Données manipulées

- Utilisateurs (nom, email, mot de passe, rôle)
- Séances sportives (titre, type, date, heure, places disponibles)
- Réservations (utilisateur, séance, date de réservation)

---

## 💻 6. Technologies utilisées

- HTML5 / CSS3 pour l’interface utilisateur
- JavaScript pour le dynamisme de l’application
- Node.js pour le backend
- Express.js pour l’API REST
- MySQL pour la base de données

---

## 📱 7. Contraintes techniques

- Interface responsive (mobile, tablette, desktop)
- Respect des règles d’accessibilité (contrastes, navigation clavier, labels)
- Sécurisation des données utilisateurs (mots de passe hashés)
- Validation des formulaires côté client et serveur

---

## 🔐 8. Contraintes de sécurité

- Authentification des utilisateurs
- Protection des routes sensibles (admin)
- Prévention des injections SQL
- Gestion sécurisée des mots de passe

---

## 🌐 9. Déploiement

L’application sera mise en ligne sur une plateforme de déploiement web afin d’être accessible via Internet.

---

## 🧪 10. Tests

- Tests fonctionnels (scénarios utilisateur)
- Tests unitaires sur les fonctions principales
- Tests de sécurité basiques (authentification, accès admin)

---

## 📌 11. Résultat attendu

Une application web complète permettant :
- la gestion de séances sportives
- la réservation en ligne
- une interface utilisateur moderne, sécurisée et responsive
